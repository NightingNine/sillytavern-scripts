# A.U.T.O 角色卡创作台

把 `A.U.T.O 预设_v2.0` 的角色卡创作流程界面化，提供分步骤创作、对话补全、结构化产物整理、世界书组装与模型连接选择。

## 依赖

- SillyTavern
- 酒馆助手（TavernHelper）
- A.U.T.O v2.0 预设 JSON（首次打开后由用户在创作台设置页导入）
- 所需正则 JSON（可单独导入；不再读取 SillyTavern 的任何正则）

## 安装

1. 在酒馆助手中新建“脚本”。
2. 将下方内容粘贴到脚本正文并保存。
3. 启用脚本，然后点击“打开 A.U.T.O 创作台”。

```javascript
import 'https://cdn.jsdelivr.net/gh/NightingNine/sillytavern-scripts@auto-card-studio-bootstrap-v3/dist/character-creation/auto-card-studio/index.js';
```

## 数据与更新

- 当前发布版本：`0.6.26`

### 清理与重新生成

- 当前步骤有对话时，可以从步骤标题区域清空本步骤对话；正式产物会转入产物历史并继续保留。
- 每个产物都可以单独删除。删除会移除该产物的全部历史版本，并让相关步骤回到可重新生成的状态。
- 删除产物后，旧的自动世界书重组方案会作废，下一次发布时自动重建。

### 输出格式配套正则

发布时若勾选 Step24 的“输出格式”，创作台会先询问是否把 9 条通用配套正则写入角色卡局部正则，再单独确认是否创建角色卡。选择不载入正则不会中止发布。

Step23 生成的“显示状态栏”正则仍按当前项目动态创建；通用正则包中不包含任何示例状态栏。
- 独立正则改用与 SillyTavern 正则扩展一致的宽松解析规则，同时兼容 `/表达式/flags` 和无外层分隔符的表达式；解析不再依赖酒馆助手 iframe 是否暴露严格解析器。
- 预设导入只保留当前 `prompt_order` 实际引用的条目，不再把 JSON 的整个 prompts 资源池当成当前预设；旧版已导入的夹带条目会在刷新后自动清理。
- “回顶 / 回底”定位按钮上移到补充输入区外边缘，不再遮挡对话区与输入区的分隔位置。
- 酒馆助手工具栏按钮名称现直接使用 `🔨` 图标字符，不再先显示“打开 A.U.T.O 创作台”后由脚本替换；旧按钮名称会在首次加载新版时自动迁移。
- 修复导入预设后渲染条目 token 估算时出现 `estimateTokens is not defined` 的问题。
- 导入预设时会自动提取并同步其中由“【SoliUmbra】预设内置正则 v2”等工具写入的 `extensions.regex_scripts`；只有预设本身不含正则时，才保留此前单独导入的正则。
- A.U.T.O 预设和回复正则现由创作台独立保存，不再绑定 SillyTavern 当前预设，也不读取全局、预设或角色正则。
- 首次使用时在“设置”页分别导入 A.U.T.O 预设 JSON 与所需正则 JSON；大型预设保存在浏览器本地数据库中，不会被复制进每个项目。
- 左侧流程显示 29 个可创作步骤；原世界书重组步骤已隐藏，并在发布角色卡与世界书时按本次勾选产物自动执行。
- 设置页右侧提供“预设 / 正则”停靠页签。展开后可查看 29 个步骤之外的预设条目及全部独立正则，并逐项启用或停用。
- `index.js` 内置自动更新器；刷新 SillyTavern 时自动检查并加载更高版本。
- 标题栏提供手动检查更新按钮，会绕过自动检查的六小时缓存；发现新版后刷新页面并自动重新打开创作台。
- 标题旁提供可重复打开的 15 步互动新手引导，按照当前真实流程讲解独立预设与正则、项目、母题、六阶段路线、核心步骤、对话、提示词、产物、模型、输出格式、AutoTask、发布和备份。
- 引导会主动打开资源抽屉、项目库与创作概览，定位 Step 1、Step 24、Step 25，并依次切换右侧“产物、设置、发布”页面；退出后恢复原步骤、折叠状态、资源抽屉、标签页及滚动位置。
- 引导支持上一步、下一步、跳过、完成以及键盘左右方向键；开启“减少动态效果”时会自动取消转场动画。
- 更新检查或新版加载失败时，会继续使用入口内置版本，不影响创作台打开。
- 工具栏入口显示为紧凑的小锤子图标，悬停可查看完整名称。
- 左侧魔法棒菜单中也提供“A.U.T.O 角色卡创作台”入口。
- 支持同时保存多个项目；点击左侧文件夹图标即可切换、新建或删除项目。
- 标题栏提供项目导出与导入按钮；导入的备份会作为独立项目加入项目库，同名时自动改名，不覆盖现有内容。
- 项目库会从当前项目卡片下方平滑展开，并避开项目名与进度区域；收起时使用对应的反向动画。
- 从旧版升级时，原来的单个本地项目会自动迁移进项目库，不需要手工导入。
- 设置页下拉菜单使用与创作台一致的自定义样式，不再显示系统原生白色菜单。
- 独立连接的模型名称改用可搜索的自定义下拉列表；获取模型成功后自动展开，也可点击箭头重新选择，并继续支持手动输入模型 ID。
- 尚未导入 A.U.T.O 预设时仍可查看和编辑项目，但生成按钮会禁用，也不会改用 SillyTavern 当前预设。
- 项目内容保存在浏览器本地存储中，更新远程脚本不会主动清空项目。
- 酒馆助手使用固定的 `auto-card-studio-bootstrap-v3` 入口。v3 直接包含手动更新与新手引导，后续更新无需再修改导入地址。
- AI 回复的显示与后续提示词上下文只通过创作台自行导入且已启用的正则处理；SillyTavern 的全局、当前角色和当前预设正则均不会参与。
- 对话中的 HTML 代码块支持“源码 / 预览”切换；预览在独立沙箱画布中渲染 CSS 和页面结构，并禁止执行其中的 JavaScript，避免影响创作台或 SillyTavern。
- 生成前可点击“查看提示词”，按实际发送顺序检查系统消息、启用的预设条目、项目上下文与本轮输入，并可一键复制完整消息队列。
- 提示词预览使用独立折叠控制：默认展开当前步骤条目，切换条目时自动收起上一条；长内容在固定阅读区内滚动，不受 SillyTavern 原生折叠样式影响。
- 同名结构化产物默认展示最新版，保留全部生成历史，可切换查看并将旧版恢复为新的当前版本。
- 左侧为 Step 1、4、7、8、24、29（原教程 Step 30）显示“核心”徽标；核心表示制卡逻辑的重要节点，不代表强制完成。
- 第 25–28 步独立归入“AutoTask 配置”，第 29 步归入“启动与交付”，便于区分后台副 AI 配置与最终开场。
- 依据原作者 Discord 教程全面修订每一步的说明弹窗、对话区初始说明、起始问题与输入示例，补全状态机、特殊语言、变量产物依赖、回复分区及 AutoTask 配置逻辑。
- Step 5“主要角色”的原点、画像与状态均作为正式产物管理；状态会参与后续上下文并写入对应世界书条目。
- 产物与提示词预览统一显示按 SillyTavern 当前 tokenizer 计算的 token 数；tokenizer 不可用时以 `≈` 标记估算值。
- 正式产物中的角色扮演玩家身份统一使用 `{{user}}`；过程说明仍可使用普通“用户”，且不会修改 A.U.T.O 预设文件。
- 产物栏按照 A.U.T.O v2.0 每个步骤的正式输出协议分别筛选；思考、设计草稿、评分和追问仍保留在对话中，但不会混入最终产物。
- 产物栏按“故事核心、主要角色、世界、叙事、变量、装配”六类筛选，也可筛选当前步骤，或按中文产物名、原始标签、步骤名称和编号搜索；筛选不会删除或改变已有产物。
- 产物条目使用中文名称显示，悬停可查看 A.U.T.O 原始标签；底层仍保留原始标签用于历史识别、后续提示词与世界书写入。
- 产物页的统计说明、阶段筛选和搜索框保持固定，只有下方产物列表独立滚动；长列表不会再带走操作控件。
- 产物列表的每一行使用真实内容高度，不再为了塞入固定区域而压扁；列表始终保留独立纵向滚动槽，展开正文后可以继续向下查看。
- 产物条目由创作台显式控制展开与收起，并显示方向箭头；“全部”及任意筛选分类都可正常查看内容。
- 浏览器缩放或可用窗口变窄时，创作台以完整三栏画布等比例适配；字体、间距和三栏比例统一缩放，不再各自触发断点重排。
- 发送给下一轮 AI 的项目上下文与产物栏共用正式产物规则：每个产物身份只发送最新版，不再包含 AI 回复中的说明、思考、评分或追问；当前步骤的用户修改要求仍会保留。
- 支持识别没有 XML 外壳的正式代码产物，包括变量 Schema、AutoTask 配置、重组方案、状态栏 HTML/正则和正式开场。
- 集合型步骤会按动态 XML 标签、变量簇或任务 ID 区分产物身份；同一身份生成新历史版本，不同身份则并列保留并一同传递给后续步骤。
- 角色卡模板变量 `{{char}}`、`{{user}}` 会在生成期间受到保护，不会被当前聊天环境提前替换为系统名或用户名。
- 模板变量保护会在酒馆助手宏替换前后各执行一次；即使嵌套宏二次生成 `{{char}}`，也会重新保护，并在提示词预览中显示为原始变量而非 `SillyTavern System`。
- “重试最新输入”替换掉的旧回复也会进入产物历史，不会继续占据对话区。
- 对话区提供“回顶”和“回底”快捷按钮，可快速定位当前步骤的第一条或最新一条消息。
- 世界书发布按产物标签读取最新版；最后一条回复未重复输出的其他标签不会丢失。
- 发布不再依赖 A.U.T.O 世界书模板；创建时可勾选要写入的正式产物，创作台会直接生成新世界书并绑定角色卡。
- 发布时，隐藏的世界书重组步骤会获得与本次勾选产物完全一致的虚拟结构报告；缺少可用 `reorg_plan` 或覆盖不完整时自动调用 AI 重新生成。
- 自动重组可控制世界书条目的合并、改名、顺序、启用状态、关键词和注入位置。
- 自动重组仍无法完整覆盖本次产物时会停止发布并提示错误，不再静默退回 A.U.T.O 默认条目结构。
- 自动重组指令作为标准用户消息直接发送，兼容无法解析后台 `user_input` 占位符的酒馆助手版本。
- Step23 的状态栏 HTML 与 `STATUSBAR_DATA` 会组合为角色卡局部正则“🕹️显示状态栏”；更新已有角色卡时保留其他局部正则。
- 产物识别以 A.U.T.O v2.0 正则中的“复制并粘贴到……”交付说明为准，包含数据盘点、条件地图、条件显示规划、状态栏与 AutoTask 配置等必要输出。
- 更新器最多每 6 小时检查一次 GitHub 内容索引，以避免频繁请求；刷新页面时若检查间隔已到便会执行检查。
- 不要同时启用旧的 SillyTavern 插件版，以免出现重复按钮或界面冲突。
