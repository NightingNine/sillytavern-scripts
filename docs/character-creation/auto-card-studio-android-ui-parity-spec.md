# A.U.T.O Android 与原脚本 1:1 UI 还原规格

## 1. 目标

Android 版在手机竖屏下复现原 A.U.T.O 酒馆脚本的视觉层级、信息密度、控件顺序、抽屉结构和主要交互。现阶段不做重新设计；后续优化必须另开需求。

## 2. 允许保留的平台差异

仅允许以下差异：

- Android 状态栏、导航栏和安全区占位；
- 软键盘弹出时的输入区避让；
- Android 原生文件选择器；
- 为触控保留必要的最小点击区域；
- 原脚本依赖 SillyTavern 的能力，在独立 App 中使用等价本地能力或明确的不可用提示。

除以上项目外，布局、间距、字号、颜色、控件顺序和交互层级均以原脚本为准。

## 3. 360 × 640 逻辑像素基准

以下尺寸不包含 Android 系统状态栏；安装到设备后只在顶部叠加安全区。

| 区域 | 原脚本基准 | Android 验收要求 |
| --- | --- | --- |
| 顶部工具栏 | 高 50px | 内容区高 50px，另加系统安全区 |
| 折叠流程轨 | 宽 56px | 固定 56px |
| 主工作区 | 宽 304px | 占用流程轨之外全部宽度 |
| 收起状态阶段头 | 高约 54px | 阶段编号、标题、必做标签同一紧凑行；目标说明在下一行 |
| 收起状态创作概览 | 高约 2px | 默认收起，只保留分隔线 |
| 对话区 | 从约 y=106 到输入区 | 自动填满剩余高度并独立滚动 |
| 底部输入区 | 高约 163px | 始终在主工作区底部可见 |
| 输入框 | 约 284 × 78px | 12px 字号，10px/12px 内边距 |
| 四项主操作 | 单行 4 等分，高 38px | 后序、提示词、生成、下一站同屏 |
| 右侧检查器 | 宽约 94vw，上限 420px | 从右侧覆盖进入 |
| 左侧完整流程 | 宽约 82vw，上限 310px | 从左侧覆盖进入 |

## 4. 主界面视觉与结构

### 4.1 顶部工具栏

顺序与原脚本一致：

1. A.U.T.O 标题；
2. 新手引导；
3. 检查更新；
4. 导出项目；
5. 导入项目；
6. 项目检查器；
7. 关闭应用。

窄屏统一使用 30–32px 图标按钮。Android 不支持脚本热更新时，“检查更新”必须给出明确提示，不能静默无响应。

### 4.2 折叠流程轨

- 阶段使用语义图标；
- 步骤使用 19px 圆形节点和两位数字；
- 当前步骤使用陶土色实心圆；
- 已完成步骤使用绿色完成态；
- 阶段间距、连接线和滚动行为与原脚本一致。

### 4.3 阶段标题与创作概览

- 默认进入时创作概览收起；
- 收起后 `PHASE 01 / 29`、步骤标题、必做标签位于同一行；
- 目标说明保持单行省略；
- 右侧保留步骤说明、清空本步和展开概览三个紧凑按钮；
- 展开概览后显示“创作母题”输入和保存操作。

### 4.4 对话空状态

- 空状态使用原脚本的居中“创作航标”结构；
- 标题、说明和三个引导问题保持原脚本字号与间距；
- 不增加额外大卡片、大标题或装饰性留白。

### 4.5 底部输入与动作

- 输入区固定在主工作区底部；
- 标签、输入框和四项动作始终同屏；
- 操作顺序为“后序、提示词、生成、下一站”；
- 生成中仅以“停止”原位替换“生成”；
- 禁用态、确认态和危险态使用原脚本颜色。

## 5. 左侧抽屉与项目库

- 完整流程抽屉顶部为“创作流程 · 29 站”和收起按钮；
- 当前项目卡、进度数字和进度条保持原脚本密度；
- 每个阶段头和步骤行使用原脚本的紧凑高度；
- 九个阶段头均可独立展开/收起，并在本次会话内保持状态；
- 项目库是流程抽屉的上一级内容，不另开全屏页面；
- 当前项目名可直接编辑；
- 项目行显示名称、确认进度、更新时间和删除入口；
- 新建项目使用虚线按钮。

## 6. 右侧检查器

### 6.1 产物

- 顶部标签固定为“产物、设置、发布”；
- 不显示独立 App 现有的大型“独立产物库”宣传头；
- 首屏依次为结构统计、说明、横向筛选、搜索和空状态；
- 空状态文案与原脚本一致；
- 产物存在时保留版本选择、上下文开关、编辑和删除能力。

### 6.2 设置

- 首屏从“模型与生成”紧凑卡片开始；
- 模型、密钥、资源、偏好与诊断分组可独立展开/收起，并在标签切换后保持状态；
- 不显示大型“模型与资源”宣传头；
- 独立 App 的模型端点、加密密钥仓和资源导入能力继续保留，但使用原脚本的卡片密度和层级；
- SillyTavern 的“使用当前连接”在独立 App 中替换为等价的本机模型模式说明。

### 6.3 发布

- 顶部为 `HANDOFF` 和交付标题；
- 角色卡名称、世界书名称、创建/导出动作保持原脚本顺序；
- 独立 App 继续导出 SillyTavern Character Card V3 JSON；
- 不显示大型重复宣传头。

## 7. 交互验收

- 展开/收起流程抽屉；
- 打开/关闭项目库并编辑项目名；
- 展开/收起创作概览；
- 打开步骤说明；
- 打开产物检查器并切换三个标签；
- 打开提示词预览；
- 触发生成或生成前置提示；
- 导入和导出项目；
- Android 返回键或遮罩可关闭当前侧栏/弹层；
- 键盘弹出后输入框和当前操作仍可见。

## 8. 验收口径

- 对照视口统一为 360 × 640 逻辑像素；
- MuMu 截图按设备密度换算后比较，不直接拿 1080 × 1920 物理像素与浏览器截图比较；
- 每个关键状态都必须同时保留原脚本与 Android 截图；
- 仅操作 ADB 设备 `127.0.0.1:7555`；
- 功能测试、TypeScript 检查、Android 构建和设备日志均通过后才算完成。
